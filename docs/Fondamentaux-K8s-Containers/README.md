# Guide : Fondamentaux Kubernetes & Conteneurs

**Auteur : Zidane Djamal**

## Prérequis Essentiels pour les Certifications OpenShift

Avant de se lancer dans les certifications OpenShift, il est indispensable de maîtriser les fondamentaux de la conteneurisation (Docker/Podman) et de Kubernetes. OpenShift étant une distribution Kubernetes enrichie par Red Hat, une compréhension solide de ces technologies de base est un facteur clé de réussite.

## Conteneurs : Docker & Podman

| Dépôt | Description |
|---|---|
| [docker-podman-complete-masterclass](https://github.com/zdmooc/docker-podman-complete-masterclass) | Cours complet Docker et Podman avec labs pratiques : création de conteneurs, volumes, réseaux, sécurité, Compose. Fonctionne indifféremment avec Docker ou Podman. Prépare à l'**EX180**. |

Ce dépôt est particulièrement important car **Podman** est l'outil de conteneurisation privilégié dans l'écosystème Red Hat. La certification **EX180** (Red Hat Certified Specialist in Containers and Kubernetes) valide ces compétences fondamentales.

## Parcours Recommandé

1. **Commencez par les conteneurs** : `docker-podman-complete-masterclass` pour maîtriser Docker et Podman.
2. **Comprenez Kubernetes en profondeur** : `demystifions-kubernetes` pour construire un cluster from scratch.
3. **Maîtrisez Helm** : `helm-masterclass` puis `helm-openshift-roadmap` pour les spécificités OpenShift.
4. **Passez aux certifications OpenShift** : Vous êtes prêt pour l'EX280 ou l'EX288.
